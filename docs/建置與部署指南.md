## 一、環境需求

| 項目 | 最低版本 | 說明 |
|------|----------|------|
| JDK | 17 | 建議使用 Eclipse Temurin 17 或 Amazon Corretto 17 |
| Gradle | 8.x（Wrapper 已包含） | 不需另外安裝，直接使用 `./gradlew` |
| 作業系統 | Windows / Linux / macOS | 均可，Windows 用 `gradlew.bat` |

> ✅ 不需要資料庫，不需要 Redis，只需要 JDK 17。

## 四、設定檔說明

`src/main/resources/application.properties`：

```properties
spring.application.name=health-decoder-api

# Swagger UI 路徑
springdoc.swagger-ui.path=/swagger-ui/index.html
springdoc.api-docs.path=/v3/api-docs
springdoc.swagger-ui.try-it-out-enabled=true

# ⚠️ 重要：decode-mqtt 解碼完成後轉送 .NET 的目標 URL
net.api.health.url=https://amicoipetwebapi-e3hpa6dkb8gngah0.japaneast-01.azurewebsites.net/api/ipetdata/health-data

# ⚠️ 重要：查詢裝置綁定寵物的動物種類（決定 SDK animalType）
net.api.animal-type.url=https://amicoipetwebapi-e3hpa6dkb8gngah0.japaneast-01.azurewebsites.net/api/pet/animal-type

# ⚠️ 重要（資安）：decode-mqtt 端點的 API Key 防護
# MQTT station 每次呼叫都必須在 Header 帶 X-Api-Key: <此值>
# 正式部署請改為強密碼並透過環境變數注入（見下方「環境變數注入」說明）
decoder.api.key=CHANGE_ME_BEFORE_DEPLOY
```

> 修改任何 `net.api.*` 或 `decoder.api.key` 後需**重啟服務**才生效。  
> 若 `net.api.health.url` 為空，`decode-mqtt` 的 .NET 轉送步驟會跳過（只解碼，不送出）。  
> 若 `decoder.api.key` 為空或為 `CHANGE_ME_BEFORE_DEPLOY`，API Key 驗證會被跳過並列印警告——**正式環境必須設定此值**。

### 環境變數注入（正式部署推薦）

不要把 API Key 明文存在 `application.properties`（避免提交到 git）。  
Spring Boot 會自動讀取環境變數，`decoder.api.key` 對應的環境變數名稱為 `DECODER_API_KEY`：

**Linux / systemd 設定方式（見第五節）：**
```ini
[Service]
Environment="DECODER_API_KEY=your-strong-secret-key"
```

**Azure App Service 設定方式：**  
前往 App Service → 設定 → 應用程式設定 → 新增：
```
名稱：DECODER_API_KEY
值：your-strong-secret-key
```

## 五、部署到 Linux 伺服器（如 Raspberry Pi / VPS）

### 步驟 1：在開發機建置 JAR

```bash
gradlew.bat build -x test
```

### 步驟 2：將 JAR 複製到伺服器

```bash
scp build/libs/health-decoder-api-0.0.1-SNAPSHOT.jar user@server-ip:/opt/health-decoder/
```

### 步驟 3：確認伺服器有 JDK 17

```bash
java -version
# 若沒有，安裝 JDK 17：
sudo apt install openjdk-17-jdk   # Ubuntu / Debian
```

### 步驟 4：建立設定檔（覆蓋 JAR 內的預設值）

在 JAR 同目錄下建立 `application.properties`，Spring Boot 會自動讀取：

```properties
# /opt/health-decoder/application.properties
net.api.health.url=https://amicoipetwebapi-e3hpa6dkb8gngah0.japaneast-01.azurewebsites.net/api/ipetdata/health-data
net.api.animal-type.url=https://amicoipetwebapi-e3hpa6dkb8gngah0.japaneast-01.azurewebsites.net/api/pet/animal-type
server.port=8080
# decoder.api.key 建議改用環境變數注入（見下方 systemd 設定）
```

> JAR 同目錄的 `application.properties` 優先級高於 JAR 內的設定。

### 步驟 5：啟動服務

```bash
cd /opt/health-decoder
java -jar health-decoder-api-0.0.1-SNAPSHOT.jar
```

### 步驟 6：背景執行（保持伺服器重啟後自動啟動）

建立 systemd 服務：

```bash
sudo nano /etc/systemd/system/health-decoder.service
```

```ini
[Unit]
Description=Health Decoder API
After=network.target

[Service]
User=ubuntu
WorkingDirectory=/opt/health-decoder
ExecStart=/usr/bin/java -jar /opt/health-decoder/health-decoder-api-0.0.1-SNAPSHOT.jar
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal
# API Key 透過環境變數注入（避免明文存在 application.properties）
Environment="DECODER_API_KEY=your-strong-secret-key"

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable health-decoder
sudo systemctl start health-decoder

# 確認狀態
sudo systemctl status health-decoder

# 查看 log
sudo journalctl -u health-decoder -f
```

## 六、部署到 Azure App Service（Java）

### 前置：Azure CLI 登入

```bash
az login
```

### 方式 A：上傳 JAR 到現有 App Service

```bash
az webapp deploy \
  --resource-group <your-rg> \
  --name <your-app-name> \
  --src-path build/libs/health-decoder-api-0.0.1-SNAPSHOT.jar \
  --type jar
```

### 方式 B：設定環境變數（取代 application.properties）

在 Azure Portal → App Service → 設定 → 應用程式設定 加入：

| 名稱 | 值 |
|------|----|
| `NET_API_HEALTH_URL` | `https://...azurewebsites.net/api/ipetdata/health-data` |
| `SERVER_PORT` | `8080` |

或用 CLI：

```bash
az webapp config appsettings set \
  --resource-group <your-rg> \
  --name <your-app-name> \
  --settings NET_API_HEALTH_URL="https://..." SERVER_PORT=8080
```

> Spring Boot 會自動將 `NET_API_HEALTH_URL` 對應到 `net.api.health.url`（底線轉點）。

## 八、常見問題

### Q1：`gradlew: Permission denied`（Linux / macOS）

```bash
chmod +x gradlew
./gradlew build
```

### Q2：啟動後馬上 crash，log 顯示 `net.api.health.url` 相關錯誤

確認 `application.properties` 中 `net.api.health.url` 有值，或設定為空字串（允許空值，解碼仍可正常運作，只是不會轉送 .NET）。

### Q3：`decode-mqtt` 回傳 `422 NO_VALID_PACKET_DECODED`

所有 32 個封包都沒有通過 SDK 解碼（`RESULT_OK`），可能原因：
- `hex_data` 格式錯誤（不是標準 16 bytes hex）
- `n` 排序後封包順序異常
- SDK buffer 已有舊資料，與新封包不相容 → 先呼叫 `DELETE /api/health/reset?deviceId=<mac>`

### Q4：RecordTime 寫進 DB 是 1975 年

MQTT payload 的 `timestamp` 欄位是裝置的**開機 uptime（ms）**，不是 Unix epoch。  
舊版程式把它當 epoch ms 轉換，導致時間錯誤（約 1975 年）。  
v1.2 起已改為直接使用**伺服器當前 UTC 時間**（`Instant.now()`）作為 RecordTime。  
若還看到異常時間，請確認 Java API 已重新部署。

### Q5：轉送 .NET 出現 `404 DeviceId 查無對應寵物`

裝置 MAC 尚未在 WebAPI 的 `Devices` 表綁定對應寵物（`PetId` 為 NULL）。需先在 APP 端完成藍牙配對，或在後台管理網站手動綁定。

### Q5：轉送 .NET 出現 `Connection refused`

`net.api.health.url` 填的是 `localhost` 但 .NET WebAPI 沒在同一台機器上。  
請改為 Azure URL：  
`https://amicoipetwebapi-e3hpa6dkb8gngah0.japaneast-01.azurewebsites.net/api/ipetdata/health-data`
